## 0. 시작 전 확인 사항 (Decision Points)

아래 항목은 개발 방향을 크게 좌우하므로, 착수 전에 선생님께서 확정해 주셔야 합니다.
현재는 20년간의 외국인 대상 체험 서비스 운영 경험을 전제로 한 **합리적 기본값(default)** 을 넣어두었습니다.

| 번호 | 결정 항목 | 기본값(가정) | 대안 |
|------|-----------|--------------|------|
| D-1 | 콘텐츠 범위 | 경주/신라 중심 문화·체험 콘텐츠 + 한국 여행 정보 | 전국 범위 / 특정 지역 한정 |
| D-2 | 지원 언어 | 영어(기본) + 한국어 | 중국어/일본어 추가 |
| D-3 | 예약·문의 기능 | 문의 폼(inquiry form) 중심, 실결제 없음 | 실시간 예약·결제 연동 |
| D-4 | 콘텐츠 편집 주체 | 비개발자(운영자)가 직접 편집 → Headless CMS 도입 | 개발자가 코드로 관리 |
| D-5 | 초기 목표 | 정보 제공 + 브랜드 신뢰 구축 + 문의 유입 | 커머스(직접 판매)까지 |
| **D-6** | **개인정보·쿠키 동의 처리** | **GDPR/한국 개인정보보호법 모두 대응(쿠키 배너 + 동의 기록)** | 최소 문구만(한국 방문자 위주일 때) |
| **D-7** | **콘텐츠 제작·감수 리소스** | **본인 초안 작성 + 원어민 감수 외주** | 전문 카피라이터 외주 / 자체 작성 |
| **D-8** | **런칭 목표 시점** | **착수 후 8~10주 내 MVP 공개** | 특정 성수기(예: 가을 관광철) 역산 |
| **D-9** | **예산 규모** | **연 운영비 저비용(무료/저가 SaaS 조합)** | CMS·이메일 유료 플랜 상향 |

> **작성자 메모**: D-1은 "20년간 외국인 대상 체험 프로그램을 개발·운영"해 오신 이력과 경주/신라 문화 스토리텔링 자산을 강점으로 보아 경주·신라 중심으로 잡았습니다. 전국 단위 콘텐츠 큐레이션을 원하시면 D-1을 변경해 주십시오.
> **신규 D-6**: 외국인(특히 EU 거주자)이 문의 폼에 개인정보를 입력하므로 GDPR 대응은 사실상 필수입니다. 상세는 15번 참조.

## 1. 프로젝트 개요

### 1.1 배경
- 한국을 방문하는 외국인 관광객은 여행 준비 단계에서 **영어로 된 신뢰할 수 있는 심층 콘텐츠**를 찾기 어렵다.
- 대형 OTA(예약 플랫폼)는 '상품'은 보여주지만 '왜 이 문화가 특별한가'라는 **맥락(스토리)** 을 전달하지 못한다.
- 20년간 축적된 현장 운영 노하우와 문화 스토리텔링 역량을 웹 콘텐츠로 전환하여 **차별화된 온라인 접점**을 만든다.

### 1.2 목적
1. 외국인 방문객에게 한국(경주·신라) 문화·체험 콘텐츠를 **읽기 쉬운 영어**로 소개한다.
2. 콘텐츠를 통해 브랜드 신뢰를 구축하고, **체험·프로그램 문의(리드)** 로 자연스럽게 연결한다.
3. 검색엔진(SEO)을 통해 여행 준비 단계의 외국인이 **자연 유입**되도록 한다.

### 1.3 핵심 가치 제안 (Value Proposition)
> "여행사가 아니라, 현지를 20년간 안내해 온 사람이 직접 들려주는 한국 문화 이야기와 체험."

### 1.4 프로젝트 범위 정의 (Scope) 〔신규〕
프로젝트 통제를 위해 **포함/제외 범위**를 명시한다.

**In-Scope (이번 프로젝트에 포함)**
- 반응형 콘텐츠 웹사이트(홈/Stories/Experiences/Guide/About/Contact)
- 영/한 2개 언어 i18n, 문의 폼 + 이메일 전송, SEO 기본 세팅, Vercel 배포

**Out-of-Scope (이번 범위 제외 — 향후 별도 과제)**
- 실시간 예약·결제 시스템, 회원·로그인, 사용자 리뷰/UGC
- 자체 백엔드 서버·DB 구축(초기엔 서버리스로 대체)
- 앱(iOS/Android) 개발, 오프라인 마케팅

### 1.5 경쟁·포지셔닝 간단 분석 〔신규〕
| 유형 | 예 | 한계 | 본 사이트의 차별점 |
|------|----|------|--------------------|
| 대형 OTA | Klook, Viator, GetYourGuide | 상품 나열, 맥락·스토리 부재 | 20년 현지 전문가의 깊이 있는 스토리텔링 |
| 공공 관광 포털 | 한국관광공사 등 | 정보는 많으나 개인화·신뢰 관계 약함 | 특정 지역(경주/신라) 전문성·인격적 신뢰 |
| 개별 블로그 | 여행 블로거 | 신뢰도·지속성·전문성 편차 | 20년 운영 실적 기반 브랜드 |

## 2. 타겟 사용자 (Personas)

| 페르소나 | 특징 | 니즈 | 핵심 시나리오 |
|----------|------|------|----------------|
| **P1. 문화 심층 여행자** | 30~50대, 개별 여행, 역사·문화 관심 高 | 표면적 관광이 아닌 맥락·의미 | 신라 역사 콘텐츠를 읽고 체험 문의 |
| **P2. MICE·단체 담당자** | 기업/기관 행사 담당, 영어 소통 | 신뢰 가능한 현지 파트너 | 체험·케이터링 역량 확인 후 견적 문의 |
| **P3. 여행 준비 단계 관광객** | 방한 1~3개월 전, 모바일 검색 | 무엇을·어디서·어떻게 | 검색 유입 → 정보 습득 → 저장/문의 |

### 설계 원칙
- **영어 우선(English-first)**: 원어민이 읽어도 자연스러운 문장. 기계번역 톤 지양.
- **모바일 우선(Mobile-first)**: 여행 중 스마트폰 열람 비중이 높음.
- **빠른 로딩·명확한 CTA**: 각 콘텐츠 하단에 '문의하기/자세히 보기' 명확 배치.

### 사용자 여정 지도 (User Journey) 〔신규〕
```
[검색/SNS 유입] → [콘텐츠 열람(Stories/Guide)] → [신뢰 형성(About)]
      → [체험 상세 확인(Experiences)] → [문의 폼 제출] → [운영자 이메일 수신·회신]
      → [재방문/저장/공유]
```
각 단계에서의 이탈 방지 장치: 관련 콘텐츠 추천, 하단 고정 CTA, 자동 응답 이메일(문의 접수 확인).

## 5. 페이지별 상세 명세

### 5.1 홈 (`/`)
- **Hero 섹션**: 임팩트 있는 대표 이미지 + 한 줄 가치 제안 + 주 CTA(Explore / Contact)
- **큐레이션 섹션**: 대표 이야기 3~4개, 대표 체험 3개 카드형 노출
- **신뢰 섹션**: 20년 이력·주요 실적(예: 대규모 행사·국제 행사 케이터링 경험)을 간결히
- **최종 CTA**: 문의 유도

### 5.2 문화 이야기 목록/상세 (`/stories`)
- 목록: 썸네일 카드 그리드 + 카테고리 필터
- 상세: 제목 / 대표이미지 / 본문(리치 텍스트·이미지 혼합) / 관련 체험 연결 / 하단 CTA
- 각 상세 페이지는 **SEO 최적화**(고유 메타, OG 이미지, 구조화 데이터)

### 5.3 체험·프로그램 (`/experiences`)
- 상세 항목: 프로그램명 / 소개 / 소요시간 / 인원 / 포함사항 / 대표 사진 / **문의 버튼**
- (D-3에 따라) 실결제 대신 **문의 폼 연동**

### 5.4 여행 가이드 (`/guide`)
- 실용 정보(교통, 계절, 추천 동선 등)를 콘텐츠 형태로 제공 → 검색 유입 핵심

### 5.5 소개 (`/about`)
- 창업 배경, 운영 철학, 문화 스토리텔링 강점, 협업/파트너십 등 **신뢰 근거** 중심 서술

### 5.6 문의 (`/contact`)
- 문의 폼(필수: 이름·이메일·메시지 / 선택: 인원·희망일·프로그램)
- **개인정보 수집·이용 동의 체크박스(필수)** + 처리방침 링크 〔신규 · D-6〕
- 스팸 방지(honeypot + reCAPTCHA/Cloudflare Turnstile 등 선택) 〔보강〕
- 전송 성공/실패/로딩 상태 명확 표시, 실패 시 재시도 안내 〔신규〕
- 지도 임베드 + 연락처 + SNS 링크

### 5.7 법적 페이지 (`/privacy`, `/terms`) 〔신규〕
- 개인정보처리방침: 수집 항목·목적·보관기간·제3자 제공·이용자 권리(GDPR: 열람·삭제·이의) 명시
- 이용약관: 콘텐츠 저작권, 면책, 문의 처리 범위

### 5.8 에러·상태 페이지 (`/404`, `/500`) 〔신규〕
- 언어별 안내, 홈/주요 콘텐츠로 복귀 링크, 브랜드 톤 유지

## 6. 콘텐츠 데이터 모델 (Content Model)

각 콘텐츠는 아래 스키마를 따른다. (CMS 또는 MDX front-matter 기준)

```yaml
# Story / Experience / Guide 공통 스키마 예시
slug: "the-story-of-wolji-pond"        # URL 식별자
title: "The Story of Wolji Pond"        # 제목 (언어별)
category: "history"                      # history | experience | food | seasonal
tags: ["silla", "gyeongju", "unesco"]
coverImage: "/images/wolji.jpg"
coverImageAlt: "Night view of Wolji Pond in Gyeongju"  # 접근성·SEO 필수 〔신규〕
excerpt: "A short 1-2 sentence summary for cards and SEO."
locale: "en"                             # en | ko
```

## 7. 기술 스택 (Tech Stack)

| 영역 | 선택 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | **Next.js (App Router) + TypeScript** | SSR/SSG로 SEO 강력, i18n 지원, 확장성 |
| 스타일링 | **Tailwind CSS** | 빠른 반응형 구현, 일관된 디자인 |
| 다국어 | **next-intl** | App Router 친화적 i18n |
| 콘텐츠 | **MDX(초기) → Headless CMS(Sanity 등)** | 초기 코드 관리 → 이후 운영자 직접 편집 (D-4) |
| 폼 처리 | **React Hook Form + 서버리스 이메일(Resend 등)** | 백엔드 없이 문의 이메일 전송 |
| 폼 검증 | **Zod** 〔신규〕 | 타입 안전한 스키마 검증, 서버·클라이언트 공용 |
| 스팸 방지 | **honeypot + Cloudflare Turnstile** 〔신규〕 | 무료·경량 봇 차단 |
| 지도 | **Google Maps 임베드** | 외국인 사용자에게 익숙함 |
| 이미지 | **next/image + Vercel 이미지 최적화** 〔신규〕 | 자동 리사이즈·WebP·지연로딩 |
| 배포 | **Vercel** | Next.js 최적, 자동 배포·CDN·프리뷰 |
| 분석 | **Google Analytics 4 / Plausible** | 방문자 행동·유입 분석 |
| 쿠키 동의 | **경량 consent 라이브러리 / 자체 구현** 〔신규〕 | GDPR 대응, GA 로딩 게이팅 |
| 에러 모니터링 | **Sentry(무료 티어)** 〔신규〕 | 런타임 오류 추적 |
| 코드 품질 | **ESLint + Prettier + TypeScript strict** 〔신규〕 | 일관성·버그 예방 |
| 테스트 | **Vitest(유닛) + Playwright(E2E)** 〔신규〕 | 핵심 플로우 회귀 방지 |

> **선택 근거 요약**: 외국인 대상 사이트의 최우선 과제는 (1) 검색 노출과 (2) 빠른 모바일 경험이므로, SSG/SSR과 i18n에 강한 Next.js를 중심에 둡니다.

### 7.1 Headless CMS 비교 (Phase 2 선택 가이드) 〔신규〕
| CMS | 장점 | 단점 | 무료 티어 | 적합도 |
|-----|------|------|-----------|--------|
| **Sanity** | 유연한 스키마, 실시간, 다국어 강함 | 학습 곡선 | 넉넉 | ★★★ (추천) |
| **Contentful** | 성숙, 안정적 | 유료 전환 빠름 | 제한적 | ★★ |
| **Strapi(셀프호스팅)** | 완전 통제, 오픈소스 | 서버 운영 부담 | 자체 호스팅 | ★★ |
| **Payload** | TS 친화, 코드-우선 | 상대적 신생 | 셀프호스팅 | ★★ |
| **Decap(구 Netlify CMS)** | Git 기반, 무료 | 대규모엔 한계 | 무료 | ★★ (초저비용) |

> 저비용·다국어·운영자 편집을 모두 고려하면 **Sanity** 우선 추천. 극저비용 유지가 목표(D-9)면 **Decap(Git 기반)** 도 검토.

## 8. 디자인 방향 (Design Direction)

- **톤**: 신뢰감 있고 문화적 깊이가 느껴지는, 절제된 프리미엄 감성
- **컬러**: 한국 전통색을 현대적으로 재해석 (예: 짙은 네이비/먹색 + 포인트 골드/단청 계열)
- **타이포**: 영문 본문 가독성 최우선(예: Inter, Source Serif 등 조합)
- **레이아웃**: 여백을 살린 콘텐츠 중심, 이미지가 주인공이 되도록
- **접근성**: 명도 대비 준수(WCAG AA), 이미지 alt 텍스트, 키보드 내비게이션

### 8.1 접근성 세부 기준 (WCAG 2.1 AA) 〔신규〕
- 본문 대비비 4.5:1 이상, 큰 텍스트 3:1 이상
- 모든 인터랙티브 요소 키보드 접근·포커스 표시(focus-visible)
- 이미지 `alt`, 아이콘 버튼 `aria-label`, 폼 라벨 연결
- 언어 속성 `<html lang>` 페이지별 정확 설정
- 동영상·움직임: prefers-reduced-motion 존중
- 스킵 네비게이션 링크 제공

### 8.2 디자인 산출물 〔신규〕
- 컬러·타이포·간격 토큰(Design Token) 정의 → Tailwind config에 반영
- 컴포넌트: 버튼/카드/폼필드/배너/네비 등 재사용 단위 정의
- (선택) 개발 착수 전 홈·상세 1종 시안 확인

> 상세 UI 설계는 개발 착수 시 Claude Code에 "frontend-design 원칙에 따라 절제된 프리미엄 톤으로" 지시하면 됩니다.

## 9. 다국어 지원 (i18n) 및 다국어 SEO

- 라우팅: `/en/...`, `/ko/...` (기본 언어 = 영어)
- 언어 전환 시 동일 콘텐츠의 해당 언어 버전으로 이동
- 메타데이터·OG 태그도 언어별로 생성
- 번역은 **원어민 감수 전제**(기계번역 그대로 게시 금지)

### 9.1 다국어 SEO 세부 〔신규〕
- **hreflang** 태그로 언어 버전 상호 연결(`en`, `ko`, `x-default`)
- 언어별 `<html lang>` 정확 설정, canonical URL 언어별 지정
- sitemap.xml에 언어 대체(alternate) 링크 포함
- 언어 자동 감지 시에도 사용자가 명시적으로 전환 가능(강제 리다이렉트 지양)
- 콘텐츠 미번역 시 fallback 정책 정의(영어 노출 + noindex 여부)

## 13. 테스트 · 품질 보증 (QA) 〔신규〕

### 13.1 테스트 범위
- **유닛(Vitest)**: 폼 검증 스키마, 유틸(날짜·i18n 헬퍼), 콘텐츠 로더
- **E2E(Playwright)**: 문의 폼 제출 성공/실패, 언어 전환, 주요 페이지 렌더
- **수동 QA 체크리스트**:
  - [ ] 모바일/데스크톱 레이아웃 깨짐 없음
  - [ ] 문의 폼 → 이메일 실제 수신 + 자동 응답 발송
  - [ ] 언어 전환 시 URL·메타·hreflang 정상
  - [ ] 404/500 페이지 정상, 잘못된 slug 처리
  - [ ] 쿠키 동의 전 GA 미로딩 확인
  - [ ] Lighthouse 4개 지표 목표 달성
  - [ ] 이미지 alt·키보드 내비게이션·대비비

### 13.2 검수 기준 (Definition of Done)
각 페이지/기능은 ① 명세 충족 ② 반응형 ③ 접근성 기본 ④ 에러 상태 처리 ⑤ Lighthouse 목표를 만족해야 완료로 본다.

| 품질 | Lighthouse 4지표 | 목표치 지속 유지 |
| 이탈 | 주요 페이지 이탈률 | 콘텐츠 개선의 근거 |

> KPI는 GA4/Plausible 이벤트(문의 제출, CTA 클릭, 언어 전환)로 측정. 문의 제출을 **핵심 전환 이벤트**로 설정.

## 15. 법적 · 컴플라이언스 (Legal & Compliance) 〔신규〕

외국인(특히 EU 거주자) 대상이므로 **개인정보·쿠키 규제 대응이 필수**입니다.

### 15.1 개인정보 처리
- **수집 항목**: 이름·이메일(필수), 인원·희망일·메시지(선택)
- **수집 목적**: 문의 응대 및 체험 상담. **목적 외 사용 금지**
- **보관 기간**: 문의 처리 후 일정 기간(예: 1년) 후 파기, 또는 동의 철회 시 삭제
- **동의**: 문의 폼에 **개인정보 수집·이용 동의 체크박스(필수)** + 처리방침 링크
- **이용자 권리(GDPR)**: 열람·정정·삭제·처리 제한·이의 제기 → 연락 방법 명시

### 15.2 쿠키·추적
- 분석 쿠키(GA 등)는 **동의 후에만 로드**(쿠키 배너로 게이팅)
- 필수 쿠키/분석 쿠키 구분, 거부 옵션 제공

### 15.3 준거 법령
- 한국 **개인정보보호법(PIPA)**
- EU **GDPR**(EU 방문자 대상), ePrivacy(쿠키)
- (미국 방문자 다수 시) 캘리포니아 CCPA 검토

### 15.4 기타
- 콘텐츠·이미지 **저작권**: 자체 촬영/구매/라이선스 명확화, 출처 표기
- 지도·폰트·서드파티 임베드의 이용약관 준수
- 사업자 정보·연락처 표기(전자상거래 확장 시 통신판매업 신고 등 별도 검토)

## 18. 콘텐츠 제작 전략 (Content Strategy) 〔신규〕

콘텐츠가 이 프로젝트의 **핵심 자산이자 크리티컬 패스**입니다.

- **런칭 최소 분량(권장)**: Stories 5건, Experiences 3~5건, Guide 3~5건
- **작성 원칙**: 20년 현장 경험의 1인칭 스토리텔링, 검색 키워드 자연 반영(E-E-A-T)
- **제작 워크플로우**: (1) 한글/영문 초안 → (2) SEO 키워드 점검 → (3) 원어민 감수 → (4) 이미지 준비(alt 포함) → (5) 발행
- **SEO 키워드**: "things to do in Gyeongju", "Silla history", "Korea cultural experience" 등 롱테일 중심
- **이미지**: 고해상 원본 확보, 웹 최적화(next/image), 저작권 확인, 대표 OG 이미지 별도 제작
- **발행 캘린더**: 격주 1건 등 지속 발행으로 SEO 신뢰·신선도 유지
